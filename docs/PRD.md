## 1. 프로젝트 개요

### 1.1 프로젝트명
개인 개발 블로그

### 1.2 목적
Notion을 CMS(Content Management System)로 활용하여 개인 기술 블로그를 자동으로 생성하고 관리하는 웹 애플리케이션입니다. Notion에서 글을 작성하면 자동으로 블로그에 반영되는 시스템을 구현합니다.

### 1.3 CMS 선택 이유
- **작성의 편의성**: Notion의 직관적인 편집 인터페이스 활용
- **자동 반영**: API를 통해 데이터 자동 동기화
- **별도 관리 서비스 불필요**: Notion을 데이터베이스로 활용하여 추가 백엔드 구축 최소화
- **비용 효율성**: Notion API는 무료 티어 제공
- **유연한 메타데이터 관리**: 카테고리, 태그, 상태 등을 Notion에서 쉽게 관리

## 2. 주요 기능

### 2.1 핵심 기능

#### 1) Notion 데이터베이스 연동
- Notion API를 통해 블로그 글 목록 자동 조회
- 변경사항 실시간 반영 (캐싱 전략 적용)

#### 2) 글 목록 페이지
- Notion 데이터베이스의 모든 발행된 글 표시
- 최신순 정렬
- 페이지네이션 또는 무한 스크롤
- 각 글의 제목, 요약(썸네일), 카테고리, 발행일 표시

#### 3) 글 상세 페이지
- 개별 글의 전체 내용 표시
- Notion 블록의 렌더링 지원 (텍스트, 제목, 이미지, 코드 블록 등)
- 메타데이터 표시 (카테고리, 태그, 발행일)
- 이전/다음 글 네비게이션

#### 4) 카테고리별 필터링
- 카테고리별 글 목록 조회
- 카테고리 페이지 자동 생성
- 동적 라우팅

#### 5) 검색 기능
- 글 제목 및 태그 기반 검색
- 실시간 검색 결과 표시

#### 6) 반응형 디자인
- 모바일, 태블릿, 데스크톱 환경 모두 지원
- Tailwind CSS를 통한 완벽한 반응형 구현

### 2.2 추가 고려사항
- **댓글 기능**: MVP 이후 추가 예정
- **SNS 공유**: 향후 구현
- **어두운 테마**: 초기 구현은 밝은 테마, 향후 확장

## 4. Notion 데이터베이스 구조

### 4.1 데이터베이스명
`Blog Posts`

### 4.2 필드 명세

| 필드명 | 타입 | 설명 | 필수 | 예시 |
|--------|------|------|------|------|
| Title | Title | 블로그 글 제목 | ✓ | "Next.js 15 새로운 기능 소개" |
| Slug | Text | URL 슬러그 (고유한 식별자) | ✓ | "nextjs-15-new-features" |
| Category | Select | 글의 카테고리 | ✓ | Frontend, Backend, DevOps, etc. |
| Tags | Multi-select | 글의 태그 (다중 선택) | ✗ | React, Next.js, TypeScript |
| Published | Date | 발행일 | ✓ | 2026-04-16 |
| Status | Select | 글의 상태 | ✓ | Draft, Published, Archived |
| Thumbnail | Files & media | 썸네일 이미지 | ✗ | (이미지 URL) |
| Summary | Text | 글의 요약 설명 | ✗ | "최신 Next.js 15의 기능들을 알아봅시다" |
| Content | Database | 글의 본문 (페이지 콘텐츠) | ✓ | (Notion 블록) |

### 4.3 필터 및 정렬
- **필터**: Status = "Published"만 표시
- **정렬**: Published 날짜 기준 내림차순 (최신순)

## 5. 화면 구성 (UI/UX)

### 5.1 홈 페이지 (`/`)
- **주요 요소**:
  - 사이트 헤더/로고
  - 최근 발행된 글 목록 (카드 형식)
  - 각 카드: 썸네일, 제목, 요약, 카테고리, 발행일
  - 페이지네이션 또는 "더보기" 버튼
  - 사이드바 또는 우측 영역에 카테고리 목록
  
### 5.2 글 상세 페이지 (`/blog/[slug]`)
- **주요 요소**:
  - 글 제목
  - 메타데이터 (작성자, 발행일, 카테고리, 태그)
  - 썸네일 이미지
  - 글 본문 (Notion 블록 렌더링)
  - 이전/다음 글 네비게이션
  - 목차 (TOC) - 선택사항

### 5.3 카테고리 페이지 (`/category/[category]`)
- **주요 요소**:
  - 카테고리명
  - 해당 카테고리의 모든 글 목록
  - 글 개수 표시
  - 페이지네이션

### 5.4 검색 결과 페이지 (`/search`)
- **주요 요소**:
  - 검색 입력창
  - 검색 결과 목록
  - "검색 결과 없음" 메시지

### 5.5 공통 요소
- **헤더**: 로고, 네비게이션 메뉴, 검색 바
- **푸터**: 저작권, 소셜 링크, 사이트맵

## 7. 구현 단계 (로드맵)

### Phase 1: 프로젝트 셋업 (1-2일)
1. **의존성 설치**
   - `@notionhq/client` 설치
   - `tailwindcss`, `shadcn/ui`, `lucide-react` 설치 및 설정

2. **환경 설정**
   - `.env.local` 파일 생성
   - Notion API 키 및 데이터베이스 ID 설정

3. **디렉토리 구조 생성**
   - `/app` - Next.js 앱 라우터
   - `/components` - React 컴포넌트
   - `/lib` - 유틸리티, API 클라이언트
   - `/public` - 정적 파일

### Phase 2: Notion API 연동 (2-3일)
1. **Notion 데이터베이스 생성**
   - Notion 워크스페이스에 `Blog Posts` 데이터베이스 생성
   - 필드 명세에 따라 속성 설정

2. **Notion API 클라이언트 구현**
   - `/lib/notion.ts` - Notion 클라이언트 초기화
   - 함수 구현:
     - `getAllPosts()` - 모든 발행된 글 조회
     - `getPostBySlug()` - 특정 글 조회
     - `getPostsByCategory()` - 카테고리별 글 조회

### Phase 3: 글 목록 페이지 (2-3일)
1. **홈 페이지 구현** (`/app/page.tsx`)
   - Notion 데이터 조회 및 표시
   - 블로그 글 카드 컴포넌트 생성
   - 페이지네이션 구현

2. **컴포넌트 개발**
   - `PostCard` - 글 카드
   - `PostList` - 글 목록
   - `Pagination` - 페이지네이션

### Phase 4: 글 상세 페이지 (3-4일)
1. **동적 라우트 구현** (`/app/blog/[slug]/page.tsx`)
   - URL 슬러그 기반 글 조회
   - 메타데이터 표시

2. **Notion 블록 렌더링**
   - 텍스트 블록
   - 제목 블록
   - 이미지 블록
   - 코드 블록
   - 리스트 블록 등

3. **컴포넌트 개발**
   - `PostDetail` - 글 상세 페이지
   - `NotionBlock` - 블록 렌더러
   - `TableOfContents` - 목차 (선택)

### Phase 5: 카테고리 필터링 (1-2일)
1. **카테고리 페이지 구현** (`/app/category/[category]/page.tsx`)
   - 동적 라우팅
   - 카테고리별 글 조회

2. **컴포넌트 개발**
   - `CategoryPage` - 카테고리 페이지
   - `CategoryList` - 카테고리 목록

### Phase 6: 스타일링 및 최적화 (2-3일)
1. **UI 개선**
   - Tailwind CSS를 활용한 기본 스타일 정의
   - shadcn/ui 컴포넌트 활용

2. **반응형 디자인**
   - 모바일 우선 접근
   - 미디어 쿼리를 통한 반응형 대응

3. **성능 최적화**
   - 이미지 최적화 (Next.js Image)
   - 캐싱 전략 (ISR, SSG 검토)
   - 번들 크기 최적화

### Phase 7: 배포 및 테스트 (1-2일)
1. **Vercel 배포 설정**
   - GitHub 저장소 연동
   - 환경 변수 설정
   - 자동 배포 구성

2. **테스트**
   - 기능 테스트
   - 크로스 브라우저 테스트
   - 반응형 디자인 테스트

3. **모니터링**
   - 배포 후 운영 모니터링

## 8. 기술적 고려사항

### 8.1 성능
- **캐싱 전략**:
  - ISR (Incremental Static Regeneration) - 일정 시간 주기로 재생성
  - 또는 요청 시점에 데이터 조회 후 캐싱
  
- **이미지 최적화**:
  - Next.js `Image` 컴포넌트 사용
  - Notion 이미지 URL 직접 활용 또는 최적화

### 8.2 보안
- **API 키 관리**: 환경 변수를 통한 서버사이드 처리
- **CORS**: Notion API는 서버사이드에서만 호출
- **콘텐츠 검증**: Notion API로부터 받는 데이터 검증

### 8.3 확장성
- **마이그레이션 대비**: 향후 다른 CMS로 변경 가능하도록 추상화된 인터페이스 설계
- **플러그인 구조**: 블록 렌더러를 확장 가능한 구조로 설계

### 8.4 SEO
- **메타 태그**: 글별 메타데이터 (title, description, og tags)
- **구조화된 데이터**: schema.org 마크업 (선택)
- **사이트맵**: 동적 사이트맵 생성

## 9. 성공 기준

1. ✓ 모든 발행된 글이 정상적으로 표시됨
2. ✓ 글 상세 페이지에서 Notion 콘텐츠가 올바르게 렌더링됨
3. ✓ 카테고리 필터링이 정상 작동
4. ✓ 모바일, 태블릿, 데스크톱에서 반응형 작동
5. ✓ Vercel에서 정상 배포 및 접속 가능
6. ✓ 페이지 로드 시간 < 3초
7. ✓ 모든 주요 브라우저에서 호환성 확인
